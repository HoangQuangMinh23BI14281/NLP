\section{Introduction}

In recent years, online recruitment platforms have grown rapidly, leading to a large amount of job description (JD) text available on the internet. These job postings usually contain important information such as required skills, salary range, job location, experience requirements, and job roles. However, this information is often written in free-form text with different writing styles and inconsistent formats. Because of this, it is time-consuming for both job seekers and recruitment systems to manually read and extract useful information from these postings. In many applications such as job recommendation, resume matching, labor market analysis, and automated recruitment systems, converting raw JD text into structured information is therefore very important.

Named Entity Recognition (NER) is a common task in Natural Language Processing (NLP) that aims to identify and classify important pieces of information in text into predefined categories. In the recruitment domain, NER can help transform unstructured job descriptions into structured fields that machines can easily process. However, this task is still challenging because job descriptions vary greatly. Different companies use different writing styles, abbreviations, incomplete sentences, and domain-specific terms. In addition, some entities such as salary or experience requirements may appear in many different formats, which makes rule-based methods difficult to design and maintain.

In this project, we study the task of \textit{information extraction from job descriptions} using Transformer-based NER models. Our goal is to automatically detect five important types of entities that are commonly used in recruitment systems: \textbf{ROLE}, \textbf{SKILL}, \textbf{LOC}, \textbf{EXP}, and \textbf{SALARY}. Instead of using manually written rules, we investigate whether pre-trained language models can learn useful contextual representations and perform well on this task. To do this, we fine-tune several Transformer models, including BERT, DistilBERT, and RoBERTa, and compare different training objectives and decoding strategies.

The motivation of this project is twofold. First, extracting structured information from job descriptions can improve the performance of recruitment systems by enabling better search, filtering, and recommendation. Second, this project provides a practical case for studying how different model architectures and loss functions perform on a domain-specific NER dataset. In particular, we explore whether techniques that address class imbalance and sequence consistency, such as Focal Loss, Dice Loss, and Conditional Random Fields (CRF), can improve performance compared to a standard cross-entropy loss.

To support this study, we build a custom dataset in JSONL format with token-level annotations following the IOB labeling scheme. The dataset includes job descriptions from different technical domains and writing styles in order to reflect the diversity found in real-world job postings. We also design an experimental pipeline that includes tokenization, label alignment, model training, and evaluation using common entity-level metrics such as precision, recall, and F1-score.

The main contributions of this project are summarized as follows:

\begin{itemize}
	\item We formulate job-description information extraction as a Transformer-based NER task with five entity types: ROLE, SKILL, LOC, EXP, and SALARY.
	\item We construct and preprocess a custom annotated dataset for extracting structured information from diverse job description text.
	\item We compare multiple pre-trained Transformer models together with several loss functions and decoding strategies, including Cross-Entropy, Focal Loss, Dice Loss, combined objectives, and CRF-based sequence modeling.
	\item We provide an empirical analysis of model performance and discuss the trade-offs between accuracy, robustness, and practical deployment.
\end{itemize}
